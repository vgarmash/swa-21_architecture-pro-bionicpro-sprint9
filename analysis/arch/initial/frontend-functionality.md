# Функциональность Frontend-приложения BionicPro

## Обзор

Frontend-приложение представляет собой одностраничное React-приложение, которое обеспечивает аутентификацию пользователей через Keycloak и предоставляет функционал для загрузки отчетов об использовании протезов.

## Структура приложения

### Точка входа

**Файл:** [`frontend/src/index.tsx`](frontend/src/index.tsx:1)

Приложение инициализируется через ReactDOM и рендерится в элемент с идентификатором `root`. Используется `React.StrictMode` для дополнительных проверок в режиме разработки.

### Главный компонент

**Файл:** [`frontend/src/App.tsx`](frontend/src/App.tsx:1)

Компонент `App` является корневым компонентом приложения и выполняет следующие функции:

- Обертывание приложения в `ReactKeycloakProvider` для интеграции с Keycloak
- Конфигурация Keycloak через переменные окружения
- Рендеринг компонента `ReportPage`

```typescript
const keycloakConfig: KeycloakConfig = {
  url: process.env.REACT_APP_KEYCLOAK_URL,
  realm: process.env.REACT_APP_KEYCLOAK_REALM||"",
  clientId: process.env.REACT_APP_KEYCLOAK_CLIENT_ID||""
};
```

### Компонент страницы отчетов

**Файл:** [`frontend/src/components/ReportPage.tsx`](frontend/src/components/ReportPage.tsx:1)

Компонент `ReportPage` является единственной страницей приложения и реализует:

1. **Аутентификация пользователей**
   - Проверка статуса аутентификации через хук `useKeycloak`
   - Отображение кнопки "Login" для неаутентифицированных пользователей
   - Перенаправление на Keycloak для аутентификации при нажатии кнопки

2. **Загрузка отчетов**
   - Кнопка "Download Report" для аутентифицированных пользователей
   - Отправка GET-запроса к API по адресу `${process.env.REACT_APP_API_URL}/reports`
   - Использование Bearer-токена из Keycloak в заголовке Authorization

3. **Состояния загрузки**
   - Индикатор загрузки "Generating Report..." во время выполнения запроса
   - Блокировка кнопки во время загрузки

4. **Обработка ошибок**
   - Отображение сообщения "Not authenticated" при отсутствии токена
   - Отображение ошибки "An error occurred" при сбое запроса
   - Красный блок с сообщением об ошибке

## Конфигурация

### Переменные окружения

**Файл:** [`frontend/.env`](frontend/.env:1)

Приложение использует следующие переменные окружения:

| Переменная | Значение | Описание |
|------------|----------|----------|
| REACT_APP_API_URL | http://localhost:8000 | URL API-сервера |
| REACT_APP_KEYCLOAK_URL | http://localhost:8080 | URL Keycloak |
| REACT_APP_KEYCLOAK_REALM | reports-realm | Имя realm в Keycloak |
| REACT_APP_KEYCLOAK_CLIENT_ID | reports-frontend | ID клиента |

## Пользовательские сценарии

### Сценарий 1: Неаутентифицированный пользователь

1. Пользователь открывает приложение
2. Приложение проверяет статус аутентификации через Keycloak
3. Если пользователь не аутентифицирован, отображается кнопка "Login"
4. При нажатии кнопки пользователь перенаправляется на страницу входа Keycloak

### Сценарий 2: Аутентифицированный пользователь

1. Пользователь открывает приложение после успешной аутентификации
2. Отображается страница "Usage Reports" с заголовком
3. Пользователь нажимает кнопку "Download Report"
4. Приложение отправляет запрос к API с Bearer-токеном
5. Во время загрузки кнопка заблокирована с текстом "Generating Report..."
6. При успешном ответе данные доступны пользователю
7. При ошибке отображается красный блок с сообщением об ошибке

## Ограничения текущей реализации

1. **Единственная страница** - приложение содержит только одну страницу (ReportPage)
2. **Отсутствует обработка ответа API** - код делает запрос, но не обрабатывает полученные данные
3. **Нет навигации** - отсутствует маршрутизация между страницами
4. **Нет CRM-интерфейса** - нет UI для управления клиентами
5. **Нет дашборда** - отсутствует отображение статистики или графиков
6. **Минимальный UI** - ограниченный набор элементов интерфейса
