# Презентация решения: Сервис отчетов BionicPRO

## Краткое содержание реализации

Команда разработала полноценный сервис отчетов для системы BionicPRO, который позволяет пользователям получать аналитические данные о работе своих протезов. Решение построено на современной архитектуре с разделением ответственности и обеспечивает высокую производительность и безопасность.

## Архитектурное решение

### Обзор системы
Согласно ТЗ подсистема Airflow размещена в отдельной папке [airflow](../airflow).


Решение состоит из трех основных компонентов:

1. **ETL Pipeline** - Apache Airflow для извлечения и трансформации данных
2. **OLAP Layer** - ClickHouse для хранения агрегированных данных
3. **API Service** - Spring Boot приложение для предоставления данных через REST API

## Техническая реализация

### ETL Pipeline
- **Расписание**: ежедневно в 02:00 UTC
- **Таски**: 4 последовательных операции
  - Извлечение данных с датчиков
  - Извлечение данных клиентов из CRM
  - Агрегация и объединение данных
  - Загрузка в ClickHouse
- **Тестирование**: 100% покрытие кода
- **Мониторинг**: Web UI Airflow для отслеживания выполнения

### OLAP Витрина
- **Движок**: ClickHouse MergeTree
- **Партиционирование**: по user_id и report_date
- **Индексация**: оптимизирована для быстрого доступа
- **Структура**: агрегированные данные с клиентской информацией

### API Сервис
- **Фреймворк**: Spring Boot 3.x
- **Безопасность**: OAuth2 с JWT валидацией через Keycloak
- **Эндпоинты**: 4 REST API
  - Получение последнего отчета
  - Получение отчета по пользователю
  - Получение истории отчетов
  - Авторизация и контроль доступа
- **Тестирование**: полное покрытие всех сценариев

## Безопасность и доступ

### Авторизация
- **Роли**: только пользователи с ролью `prothetic_user`
- **JWT токены**: валидация через Keycloak
- **Проверка доступа**: пользователи могут видеть только свои данные
- **Защита**: предотвращение доступа к данным других пользователей

### Ограничение доступа
- Проверка в сервисном слое
- Выбрасывание исключений при нарушении
- Логирование всех попыток доступа
- Тесты на защиту от несанкционированного доступа

- **Интеграция**: через REST API

### Опыт пользователя
- Простая авторизация

- Четкое отображение статистики
- Интуитивно понятный интерфейс

## Производительность

### Оптимизации
- **ClickHouse**: оптимизирован для аналитических запросов
- **Агрегация**: данные предварительно агрегированы
- **Кэширование**: не требуется благодаря OLAP архитектуре
- **Параллелизация**: ETL pipeline выполняется параллельно

### Масштабирование
- **Горизонтальное масштабирование**: ClickHouse поддерживает кластеризацию
- **Вертикальное масштабирование**: Spring Boot приложение stateless
- **ETL**: может быть запущен на нескольких узлах

## Развертывание

### Инфраструктура
- **Контейнеризация**: Docker Compose
- **Сервисы**: 8 контейнеров
- **Сеть**: изолированная сеть
- **Хранилище**: volume для данных

### Запуск
```bash
# Клонирование репозитория
git clone <repo-url>

# Копирование конфигурации
cp .env.example .env

# Запуск сервисов
docker-compose up -d

# Проверка статуса
docker-compose ps
```
