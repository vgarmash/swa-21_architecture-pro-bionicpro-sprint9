## 3. Развёртывание Reports API Service

### 3.1 Сборка
```bash
# Сборка JAR файла
cd bionicpro-reports
mvn clean package
```

### 3.2 Docker сборка
```bash
# Сборка Docker образа
docker build -t bionicpro-reports:latest .
```

### 3.3 Запуск
```bash
# Добавление в docker-compose и запуск
docker-compose up -d bionicpro-reports

# Проверка логов
docker-compose logs -f bionicpro-reports
```

### 3.4 Проверка здоровья
```bash
# Проверка эндпоинта health
curl http://localhost:8081/actuator/health
```

## 8. Сценарии использования

### 8.1 Полный сценарий
1. Пользователь открывает http://localhost:3000
2. Перенаправляется на Keycloak для входа
3. После входа видит страницу отчётов
4. Нажимает "Download Report"
5. Получает JSON с данными о своём протезе

### 8.2 Проверка ETL
1. Данные поступают в sensors-db
2. Airflow DAG запускается по расписанию
3. Данные агрегируются и загружаются в ClickHouse
4. API возвращает актуальные данные
