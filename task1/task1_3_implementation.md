# Реализация изменений по задаче: BionicPRO Auth Service

## 1. Введение

Данная задача посвящена созданию нового бэкенд-сервиса `bionicpro-auth` для централизованного управления аутентификацией и сессиями пользователей. Вместо реализации логики управления токенами на фронтенде, была перенесена в отдельный сервис, что повышает безопасность и упрощает поддержку системы.

## 2. Архитектура решения

### 2.1 Общая архитектура сервиса

Новый сервис `bionicpro-auth` реализует следующую архитектуру:

- **Технологический стек**: Spring Boot 3.x, Spring Security 6.x, OAuth2 Resource Server
- **Конфигурация**: Внедрение зависимостей через Spring DI
- **Репозитории**: In-Memory хранение сессий в ConcurrentHashMap
- **Безопасность**: JWT токены с HTTP-only cookie для защиты от XSS атак

## 3. Реализация ключевых компонентов

### 3.1 Интеграция с Keycloak

Сервис интегрирован с Keycloak для управления аутентификацией:

- **Конфигурация**: Настройка подключения к Keycloak через `application.yml`
- **Токены**: Используются access и refresh токены с настройками в Keycloak
- **PKCE**: Поддержка протокола PKCE для защиты от атак
- **OAuth2**: Использование OAuth2 Resource Server для проверки токенов

### 3.2 Управление токенами (access и refresh)

Внедрена логика управления токенами:

- **Access Token**: Короткоживущий (2 минуты) для повышения безопасности
- **Refresh Token**: Долгоживущий (1 час) для автоматического обновления
- **Обновление**: Автоматическое обновление access токенов при истечении срока
- **Валидация**: Проверка токенов на соответствие JWT стандартам

### 3.3 Хранение в оперативной памяти

Используется in-memory репозиторий для хранения сессий:

- **Структура данных**: `ConcurrentHashMap` для обеспечения потокобезопасности
- **Сессии**: Каждая сессия содержит данные о пользователе, токенах и времени жизни
- **Управление**: Создание, обновление, удаление сессий

### 3.4 Сессионные cookie

Реализована безопасная система сессионных cookie:

- **Имя**: `SESSION_ID` 
- **Параметры**: `httpOnly: true`, `secure: false`, `sameSite: Lax`
- **Время жизни**: 1 час
- **Защита**: Защита от XSS и CSRF атак

## 4. Изменения в фронтенде

В результате реализации были внесены следующие изменения в фронтенд:

- **Удалены зависимости Keycloak**: Удалены `@react-keycloak/web` и `keycloak-js`
- **Изменена логика аутентификации**: Все операции теперь происходят через `bionicpro-auth`
- **HTTP клиент**: Обновлена настройка клиента с `withCredentials: true`
- **Удалены утилиты PKCE**: `pkceUtils.ts` и `keycloakService.ts` удалены

## 5. Настройки Keycloak

Настройки Keycloak обновлены для корректной работы с новым сервисом:

- **Таймауты**: `access.token.lifespan` = 120 секунд, `refresh.token.lifespan` = 3600 секунд
- **PKCE**: `pkce.code.challenge.method` = S256
- **Конфигурация клиента**: Обновлены настройки для `reports-frontend`

## 6. Документация по API

### 6.1 Основные эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/session/validate` | Проверка сессии |
| GET | `/session/info` | Информация о сессии |
| GET | `/auth/login` | Инициация входа |
| POST | `/auth/callback` | Обработка callback |
| POST | `/auth/refresh` | Обновление токенов |
| POST | `/auth/logout` | Выход |

### 6.2 Ответы

Все API возвращает JSON данные с соответствующими статусами и полями, согласно определенным моделям.

## 7. Заключение

Реализация нового сервиса `bionicpro-auth` успешно завершена. Система обеспечивает:

1. **Повышенную безопасность** за счет использования HTTP-only cookie и короткоживущих токенов
2. **Централизованное управление** сессиями в одном месте
3. **Удобство поддержки** за счет четкого разделения ответственности
4. **Автоматическое обновление** токенов без вмешательства пользователя

Этот подход соответствует современным стандартам безопасности веб-приложений и обеспечивает надежную аутентификацию пользователей.
